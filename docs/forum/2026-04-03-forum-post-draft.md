正文草稿：

大家好，我最近在做一个偏小而专的 `CalculiX` 诊断工具，目标不是替代完整 CAE 软件，而是先解决一个很具体的问题：

**用户遇到报错后，不知道先看哪里、先改什么。**

所以我现在优先做的是：

- 常见报错识别
- 修复建议
- 少量低风险自动修复
- 修复后验证状态

当前这个阶段我没有追求“大而全”，而是先把规则层做稳。  
截至 **2026-04-03**，当前版本已经有：

- 11 个回归样本 case
- 10 类已纳入回归的问题族
- 统一的诊断摘要 / 优先级输出
- 3 类白名单自动修复
- 修复后验证状态

目前已覆盖的问题类型包括：

- 输入关键词/语法错误
- 材料缺少 `*ELASTIC`
- 边界条件缺失或明显不足
- 刚体运动/欠约束
- 接触定义缺项
- 收敛失败
- 载荷没有真正传到结构
- 畸变单元
- 结果文件缺失
- 单位量级异常

现在的输出大概是这种风格：

```text
诊断摘要
- 严重问题：1
- 警告问题：1
- 最优先问题：Input Syntax Issue
- 首步建议：检查 INP 文件中的卡片拼写，确保与 CalculiX 支持的关键词一致

[P1] [input_syntax] 检测到无效 INP 关键词，可能是拼写错误或版本不兼容
-> 检查 INP 文件中的卡片拼写，确保与 CalculiX 支持的关键词一致
```

白名单自动修复目前只做 3 类低风险问题：

- 缺少 `*ELASTIC`
- 缺少 `*STEP`
- `*STATIC` 初始增量过大

不会自动改这些内容：

- 载荷大小
- 约束数值
- 接触参数
- 真实材料参数
- 网格策略

我故意把边界收得很窄，因为我不想把一个物理上错误的模型，修成“数值上勉强能跑”的模型。

例如，缺少 `*ELASTIC` 的情况，现在可以做到：

```text
changes=Added *ELASTIC placeholder for material STEEL
verification=passed
notes=material_missing_elastic: *ELASTIC block present
```

我现在最想听的是大家的真实使用反馈，而不是夸不夸这个工具。

如果你用过 `CalculiX`，很想请教 4 个问题：

1. 你最常见的 3 类报错是什么？
2. 你更需要“定位问题”，还是“给出修复建议”？
3. 你能接受工具自动修改哪些内容，哪些一定要人工确认？
4. 如果愿意试用，你最想先拿哪类 case 测一下？

如果有人愿意，我也很想收集一些匿名的报错样本，继续补回归 case。
